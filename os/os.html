OAuth 2.0 with PingFederate

OAuth 2.0 for Apps

  • Using OAuth 2.0 for Apps is similar to using OAuth 2.0 for websites.
  • The difference compared with Web-based clients is that the app needs to open the system browser (embedded webviews are strongly discouraged) and supply a local Redirect URI on which it can receive the authorization code.
  • Unlike with web clients, the OAuth 2.0 client secret (if used) is assumed not to be confidential.
